Schema Markup Guide

Structured data that tells Google exactly what Mad Hudson products are — and who made them

TL;DR

  • Schema markup is invisible code that tells search engines what your content means — It's not for humans to read, it's for Google's algorithms to parse.
  • It enables rich search results: star ratings, price displays, "limited edition" badges right in the search listing. This is how you stand out before anyone clicks.
  • Zero of 11 competitors have artist/creator schema — They all have basic Product schema (Shopify default), but no one connects products to artists in structured data. First-mover advantage is wide open.
  • One JSON-LD template in the product theme file auto-populates for every product — You add it once, and Shopify's Liquid variables fill in the product-specific details automatically.
  • Typical CTR lift from rich results: 20-40% — When your listing shows stars, price, and "Limited Edition," people click.

Bottom line: This is a one-time template edit that compounds forever. The competitive gap is wide open.

What JSON-LD Schema Is

▼

Schema markup is a block of code added to the <head> section of a page that describes the content in a structured way. It's written in JSON-LD (JavaScript Object Notation for Linked Data) and it's completely invisible to visitors.

Think of it like the nutrition label on food packaging — customers might not read it, but it's how the system categorizes and presents the product. Google reads schema markup to decide how to display your pages in search results.

Without Schema

Plain text link, generic description, no visual features. Your listing looks like everyone else's.

With Schema

Star ratings, price, availability, "limited edition" badge next to your listing. Stands out visually before anyone clicks.

The Analogy

Schema markup is like engraving "Swiss Made" on a caseback — it authenticates and categorizes. Visitors don't see it. But Google uses it to decide how to present your products.

Answer engines (ChatGPT, Perplexity) also read schema markup when browsing pages for recommendations. If your schema says "creator": "Artist Name", the model can cite that directly: "The CD Watch is designed by [Artist Name], based in Chicago."

Current Schema (Shopify Default)

▼

Shopify automatically adds basic Product schema to every product page. This is the minimum — name, price, availability, image URL.

What Shopify Auto-Generates (Simplified)

{
  "@context": "https://schema.org",
  "@type": "Product",
  "name": "CD Watch",
  "image": "https://cdn.shopify.com/...",
  "offers": {
    "@type": "Offer",
    "price": "215.00",
    "priceCurrency": "USD",
    "availability": "https://schema.org/InStock"
  }
}

What's Missing

  • No artist/creator info — The product isn't connected to a Person entity. Google can't surface "designed by [Artist]" in results.
  • No edition details — No way to signal that this is a limited edition.
  • No brand description — Just the brand name, no context about what Mad Hudson is.
  • No materials — No structured data about case, strap, or movement specs.
  • No review data — Even if you have reviews, they're not structured for rich results.

Status: Shopify's default schema is functional but minimal. It works for basic listings, but it doesn't communicate what makes Mad Hudson different. Competitors with basic Shopify themes have the exact same schema. No differentiation.

Complete JSON-LD Template

▼

This is the ready-to-paste template for your product theme file. Copy this entire block and add it to your product template. The Liquid variables (the parts in curly braces) auto-populate based on the product being viewed.

Where this goes: In your Shopify theme's product.liquid file (or the product template section in newer themes), inside the <head> section or just before </body>.

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Product",
  "name": "{{ product.title }}",
  "description": "{{ product.description | strip_html | truncate: 500 }}",
  "image": "{{ product.featured_image | image_url: width: 1200 }}",
  "sku": "{{ product.selected_or_first_available_variant.sku }}",
  "brand": {
    "@type": "Brand",
    "name": "Mad Hudson Watches",
    "description": "Artist collaboration watches from Chicago. Each watch is designed by a named, credited independent artist.",
    "url": "https://madhudsonwatches.com"
  },
  "creator": {
    "@type": "Person",
    "name": "[ARTIST NAME]",
    "description": "[ARTIST BIO - 1-2 sentences]",
    "address": {
      "@type": "PostalAddress",
      "addressLocality": "[ARTIST CITY]",
      "addressRegion": "[ARTIST STATE]"
    }
  },
  "offers": {
    "@type": "Offer",
    "price": "{{ product.price | money_without_currency }}",
    "priceCurrency": "USD",
    "availability": "{% if product.available %}https://schema.org/InStock{% else %}https://schema.org/OutOfStock{% endif %}",
    "url": "{{ shop.url }}{{ product.url }}",
    "seller": {
      "@type": "Organization",
      "name": "Mad Hudson Watches"
    }
  },
  "additionalProperty": [
    {
      "@type": "PropertyValue",
      "name": "Movement",
      "value": "Miyota Quartz"
    },
    {
      "@type": "PropertyValue",
      "name": "Case Material",
      "value": "Stainless Steel"
    },
    {
      "@type": "PropertyValue",
      "name": "Case Size",
      "value": "40mm"
    },
    {
      "@type": "PropertyValue",
      "name": "Water Resistance",
      "value": "3ATM"
    },
    {
      "@type": "PropertyValue",
      "name": "Limited Edition",
      "value": "true"
    }
  ]
}
</script>

About the Template Variables

Auto-populating fields: product.title, product.description, product.featured_image, product.price, and product.available all pull from Shopify's product data automatically. You don't need to change these.

Per-product fields: The creator section (artist name, bio, city, state) needs to be customized per product. The best way to do this is with Shopify metafields (explained in the next section). If you don't want to use metafields, you can hardcode per product using Liquid conditionals based on product.handle.

Example: Hardcoding Per Product (Alternative)

{% if product.handle == 'cd-watch' %}
  "creator": {
    "@type": "Person",
    "name": "Artist Name Here",
    "description": "Chicago-based visual artist known for abstract geometric patterns.",
    "address": {
      "@type": "PostalAddress",
      "addressLocality": "Chicago",
      "addressRegion": "IL"
    }
  },
{% elsif product.handle == 'another-watch' %}
  "creator": {
    "@type": "Person",
    "name": "Another Artist",
    ...
  },
{% endif %}

This works, but it gets unwieldy with many products. Metafields are cleaner.

Implementation in Shopify

▼

Step-by-step instructions for adding the schema template to your Shopify theme.

Step 1: Duplicate Your Theme (Safety First)

Always backup before editing templates. In Shopify admin:

  1. Go to Online Store → Themes
  2. Find your current live theme
  3. Click Actions → Duplicate
  4. Rename the duplicate to "[Theme Name] - Backup [Date]"

If anything breaks, you can revert to this copy instantly.

Step 2: Edit the Product Template

  1. Go to Online Store → Themes → Actions → Edit code
  2. In the left sidebar, find product.liquid (or look in the sections/ folder for product-template.liquid in newer themes)
  3. Scroll to the end of the file, just before the closing </body> tag
  4. Paste the entire JSON-LD template from the section above
  5. Click Save

Step 3: Set Up Metafields for Artist Data

Metafields let you store custom data per product without hardcoding it in the theme. This is the cleanest approach for artist info.

  1. Go to Settings → Custom data → Products
  2. Click Add definition
  3. Create the following metafields:
    • Namespace: custom
    • Name: artist_name (Type: Single line text)
    • Name: artist_bio (Type: Multi-line text)
    • Name: artist_city (Type: Single line text)
    • Name: artist_state (Type: Single line text)
  4. Save each metafield definition

Step 4: Populate Metafields for Each Product

  1. Go to Products in Shopify admin
  2. Open a product (e.g., CD Watch)
  3. Scroll down to the Metafields section
  4. Fill in:
    • artist_name: "Artist Full Name"
    • artist_bio: "1-2 sentence bio about the artist's style and background"
    • artist_city: "Chicago"
    • artist_state: "IL"
  5. Save the product
  6. Repeat for all products

Step 5: Update the Template to Use Metafields

Replace the hardcoded [ARTIST NAME] placeholders in the JSON-LD template with metafield references:

"creator": {
  "@type": "Person",
  "name": "{{ product.metafields.custom.artist_name }}",
  "description": "{{ product.metafields.custom.artist_bio }}",
  "address": {
    "@type": "PostalAddress",
    "addressLocality": "{{ product.metafields.custom.artist_city }}",
    "addressRegion": "{{ product.metafields.custom.artist_state }}"
  }
},

Step 6: Test the Schema

Save the template, then test on a live product page (instructions in the next section).

Testing Your Schema

▼

After adding the schema template, you need to verify that Google can read it correctly. Use these three tools:

1. Google Rich Results Test

URL: https://search.google.com/test/rich-results

  1. Paste your product page URL
  2. Click "Test URL"
  3. Wait for the scan to complete
  4. Check that:
    • Product entity is detected
    • Creator info is present (look for "Person" entity nested under Product)
    • No critical errors

What you're looking for: "Product" badge in green with no errors. Click "View tested page" to see exactly what Google parsed. If creator info is missing, double-check your metafield names.

2. Schema.org Validator

URL: https://validator.schema.org/

  1. Copy the JSON-LD block directly from your page source (view-source in browser, Ctrl+F for "application/ld+json")
  2. Paste into the validator
  3. Click "Validate"
  4. Check for syntax errors (missing commas, mismatched brackets)

This validator is stricter than Google's test — it catches formatting errors that might not break rich results but could cause issues later.

3. Google Search Console (Long-Term Monitoring)

Once your schema is live, Google Search Console tracks how it performs over time.

  1. Go to Search Console → Enhancements → Products
  2. Wait 1-2 weeks for Google to re-crawl your product pages
  3. Check for:
    • Number of valid product pages
    • Rich result impressions (how often your listings show with stars/price)
    • Errors or warnings

Patience required: Rich results don't appear instantly. It can take 2-4 weeks for Google to re-index your pages and start showing enhanced listings. The schema is working even if you don't see stars immediately.

What Success Looks Like

  • In Rich Results Test: Green "Product" badge, creator name visible in parsed data
  • In Schema.org Validator: No errors, all fields present
  • In Search Console: Product pages marked "Valid," rich result impressions increasing over time
  • In live search: Eventually, your product listings show price, availability, and star ratings (once you implement reviews)
