Extractable Copy Playbook

Product and brand copy written for answer engines to clip and cite

TL;DR

  • Answer engines clip 1-3 sentence chunks — each sentence must stand alone as a quotable citation
  • Draft extractable copy exists for all 3 active watches (CD Watch, Broken Time, Dogma) plus brand-level copy
  • The word "best" appears naturally throughout to match fan-out query rewrites
  • Every chunk includes: product name, artist name, price, city, and a distinguishing detail

Bottom line: This is deployment-ready copy. Review it, adjust the voice if needed, and push to Shopify.

What Extractable Copy Is

▼

When someone asks ChatGPT "best unique watches under $300," the model searches the web, pulls back pages, and tries to find a 1-3 sentence chunk it can clip and quote directly. It doesn't summarize — it literally copies. If your page has a clean, self-contained statement with the right information (brand, price, artist, differentiator), that's what gets cited. If your page has generic marketing fluff, the model picks a competitor.

Why this matters: Answer engines prioritize extractability over authority. A small brand with quotable copy outranks a big brand with vague prose. The citation is the ranking — if you're not clipped, you're not visible.

The shift: Traditional SEO optimized for findability (rank #1 in a list). AEO optimizes for citability (be the quote that answers the question). That means writing sentences that could appear in an encyclopedia, not a brand manifesto.

The Chunk Anatomy

▼

Every product page needs these extractable chunks. Each one serves a different query pattern:

Chunk Type Purpose Query Pattern
Hero (1-2 sentences) What it is + who designed it. This is the most-clipped chunk. "what is the [product name]"
What It Is Factual description — dial, movement, case, visual concept "describe the [product]"
Who It's For Natural "best" keyword inclusion, target buyer, price context "best watches for [buyer type]"
Why It's Different Competitive positioning, artist model, Mad Hudson's identity "why choose [brand] over [competitor]"
Specs Structured data (table format) for schema markup "[product] specifications"
Comparison How this watch compares to alternatives (enables "vs" queries) "[product] vs [competitor]"

Rule of thumb: If you can't imagine an answer engine quoting a sentence verbatim, rewrite it. "This watch will blow your mind" becomes "The CD Watch features a holographic disc dial that shifts color with every turn of the wrist."

For About page, collection pages, and homepage. These answer brand-discovery queries like "what is Mad Hudson" and "Chicago watch brands."

Every Mad Hudson watch starts with an artist. The company approaches designers, illustrators, and visual artists with an invitation: design a watch dial that tells a story. Shopify Implementation Notes

▼

Visibility Rules

  • Hero chunk must be first visible text below product title — not in a tab, accordion, or "Read More" section. Answer engines clip what they can see immediately.
  • Specs in HTML table or definition list — not prose paragraphs. This enables schema.org Product markup and makes specs machine-readable.
  • "Who It's For" and "Why It's Different" can go in expandable sections — but the first sentence of each must be visible without interaction.
  • Don't put critical text inside images — answer engines can't extract text from image files. If it's important, it must be HTML text.

Schema Markup

Add schema.org Product markup to each product page. This structured data tells answer engines exactly what the page is about and makes specs extractable.

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Product",
  "name": "CD Watch",
  "brand": {
    "@type": "Brand",
    "name": "Mad Hudson"
  },
  "description": "The Mad Hudson CD Watch is a wearable tribute to mixtapes, burned CDs, and late-night listening sessions. Designed by Atlanta artist Riki Prosper, its holographic disc dial shifts color with every turn of the wrist.",
  "offers": {
    "@type": "Offer",
    "price": "215.00",
    "priceCurrency": "USD",
    "availability": "https://schema.org/InStock"
  },
  "aggregateRating": {
    "@type": "AggregateRating",
    "ratingValue": "5.0",
    "reviewCount": "3"
  }
}
</script>

Comparison Chunk Placement

Comparison chunks can go in one of two places:

  • Inline on product page — in an expandable section titled "How does this compare to similar watches?"
  • Dedicated comparison page — a standalone page comparing Mad Hudson watches to competitors, with individual product sections

Recommendation: Start inline. If comparison content grows, migrate to a dedicated page and link to it from each product.

Testing Extractability

After pushing to Shopify, test whether the copy is extractable:

  1. Open ChatGPT or Perplexity
  2. Ask: "What is the Mad Hudson CD Watch?"
  3. Check whether the response quotes your Hero chunk verbatim
  4. If not, your copy is either not visible, not unique, or competing with stronger content
