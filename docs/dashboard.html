Meta Description Opportunities

Most product pages have no custom meta description — Shopify auto-generates one from the first ~160 characters of the product description. Since current descriptions are only 1-2 sentences of generic copy, the auto-generated meta descriptions are thin and keyword-poor.

Example of current auto-generated meta: "A beautiful timepiece from Mad Hudson Watches." This tells Google nothing about the artist, the edition size, or why someone would buy it.
A meta description is the display card in a watch case at a store. It's what people read before they pick up the watch. Right now, yours say "A beautiful timepiece." Imagine a display card that just said that.

Recommended meta description template:

[Watch Name] — a limited edition watch designed by [Artist Name] for Mad Hudson. [One sentence about the design]. [Edition size] pieces. Free shipping.

How to change: Same "Search engine listing preview" section in Shopify admin. Edit the "Description" field. Aim for 150-160 characters. Include artist name, edition info, and "limited edition" or "artist collaboration."

Heading Hierarchy (H1-H6)

Shopify themes sometimes misuse heading tags for visual styling rather than semantic structure. Common opportunities to check:

  • Multiple H1 tags per page: Each page should have exactly one H1. If the theme uses H1 for both the logo and the product name, the product name H1 is diluted.
  • Skipped heading levels: Going from H1 to H3 (skipping H2) confuses search engine crawlers about content hierarchy.
  • H2 sections ready to add: Product pages should use H2 for sections like "About the Artist," "Specifications," "Edition Details." Currently these sections don't exist in the content.

How to check: Right-click any product page → View Page Source → search for <h1, <h2, etc. Count how many H1 tags appear. Verify the hierarchy doesn't skip levels.

URL Structure

Shopify handles URL structure reasonably by default (/products/[handle], /collections/[handle]). The main opportunity is ensuring product URL handles include the artist name where they don't already. URL handles can be edited in Shopify without breaking existing links if redirects are set up.

Visible result: Google your watch names and the text below the link shows your custom description — artist names, edition sizes, your language — instead of auto-generated copy.

Alt Text
The single highest-impact invisible improvement

Product images have empty or auto-generated alt attributes. Google Image Search, screen readers, and rich snippets can't see them. Adding descriptive alt text changes nothing visually.

Think of alt text like the movement inside a watch case. Customers don't see it directly — but without it, nothing works for search engines. Google reads alt text the way a watchmaker reads a caliber number.

What Alt Text Should Include

Every product image needs a descriptive alt attribute that tells search engines what's in the image. Shopify makes this easy — each image in the product editor has an "Alt text" field.

Image Type Current Alt Text (typical) Recommended Alt Text
Product hero shot "" (empty) or auto-generated filename [Watch Name] by [Artist] — [brief visual description, e.g. "black dial with hand-painted coral reef motif"] — Mad Hudson Watches
Detail / dial close-up "" (empty) Close-up of [Watch Name] dial showing [specific design element] by artist [Name]
Wrist shot / lifestyle "" (empty) [Watch Name] artist collaboration watch worn on wrist — limited edition by [Artist]
Artist portrait (if present) N/A (doesn't exist yet) Artist [Name], designer of [Watch Name] for Mad Hudson Watches

Image Filename Optimization

Shopify doesn't let you rename uploaded image files directly, but new uploads should follow a descriptive naming convention before upload:

  • Current pattern: IMG_4532.jpg, product-photo-1.png, Screenshot 2025-12-03.png
  • Recommended pattern: mad-hudson-[watch-name]-[artist]-[view].jpg
  • Example: mad-hudson-coral-reef-watch-jane-doe-front.jpg

For existing images that can't be renamed, the alt text fix is more impactful and should be prioritized.

Image Size and Format

Shopify's CDN handles image compression and WebP conversion automatically. No action needed on file format. However, uploading oversized originals (5000px+) can slow down the admin interface. Aim for 2048px on the longest edge for product photos — Shopify will generate all needed thumbnails.

Untapped traffic source: "Artist collaboration watch," "limited edition art watch," "[artist name] watch" — all image-searchable queries. Proper alt text makes product images appear in Google Image results. Zero visual changes.
After this fix: Product images start appearing in Google Image results for "artist collaboration watch" and "[artist name] watch." Expect image-sourced traffic in Shopify analytics within 2-4 weeks.

No competitor has artist creator schema

Structuring this data first shapes how Google categorizes artist-collaboration watches
▼
What is schema markup? It's invisible code added to the page that tells Google exactly what the content means — who made it, what it costs, whether it's in stock. It enables rich search results (star ratings, price displays, "limited edition" badges in search). Visitors never see it. It lives in the page's <head> section as JSON-LD. Schema markup is like engraving "Swiss Made" on the caseback — it's information that authenticates and categorizes. Visitors never see it. But Google uses it to decide how to present your products.

Current Schema (Shopify Default)

Shopify automatically adds basic Product schema to every product page: name, price, availability, image URL. This is the minimum. It doesn't include artist/creator information, edition details, or review data.

Recommended Schema Additions

Schema Property Currently Present? Value Impact
creator (Person) No Artist name, URL, description Connects products to artist identity in Google's Knowledge Graph. Enables "designed by [Artist]" in search results.
isLimitedEdition No true/false Signals scarcity. Google can surface this in shopping results and product panels.
material No Case material, strap material Enables filtering in Google Shopping. Answers spec-comparison queries directly in search.
brand.description No "Artist collaboration watches from Chicago" Enriches the brand entity in search. Supports Knowledge Panel development.
review / aggregateRating No Star ratings from customer reviews Enables star rating display in search results. Significant click-through-rate improvement (typically 15-25% increase).

Implementation Approach

Schema markup is added by editing the Shopify theme's product.liquid template (or the equivalent in a newer JSON-based theme). The JSON-LD block goes in the <head> section. It needs to be templated once — then every product page automatically pulls the right data from Shopify's product fields.

None of the competitors (Mr Jones, Swatch x Art, Baltic, Brew) have artist creator schema. This is an uncontested opportunity. Being first to structure this data gives Mad Hudson a head start in how Google represents artist-collaboration watches.

Competitive advantage: Schema markup for artist attribution is something zero competitors have implemented. This positions Mad Hudson to define how Google categorizes artist collaboration watches — an indexing advantage that compounds over time.
Over time: Richer Google results — star ratings, price displays, "limited edition" badges next to your listings. More complete and trustworthy than competitors'.

How Answer Engines Work

When someone asks ChatGPT "best artist collaboration watches" or "unique watches under $300," the model doesn't search with that exact query. It rewrites the prompt into 3-4 shorter queries — a process called "query fan-out." Then it searches the web, pulls back results, and runs them through three gates:

  1. Fetchable — Can the answer engine access the page? JavaScript-rendered sites and crawl-blocked content get bypassed entirely. LLMs are increasingly sensitive to the compute cost of parsing complex pages.
  2. Choosable — Does the page contain the answer? The model picks 2-3 pages per fan-out query that look most relevant.
  3. Extractable — Can the model clip a clean 1-3 sentence chunk? Answer engines don't summarize — they literally clip and quote. If the answer is buried in 4 paragraphs of marketing fluff, it won't get extracted. A competitor with 4 clean sentences wins.

  2. Verify site is fetchable by AI crawlers — Check robots.txt for AI bot blocking. Verify product content renders in HTML source, not via client-side JavaScript. LLMs skip JS-heavy pages.
  3. Seed "best" language naturally — Answer engines flatten all queries ("top," "recommended," "most popular") to "best [category] 2026." The word "best" should appear naturally in headings and copy: "one of the best artist-collaboration watches under $300."
  4. Build entity presence across multiple sources — Mad Hudson needs mentions on Reddit (r/watches, r/microbrands), niche watch blogs (Worn & Wound, Fratello), and review platforms. Entity validation requires confirmation from 3+ independent sources.
  5. Target niche publications, not tier-1 press — 97.4% of answer engine citations come from trade publications, blogs, and Substacks. Pitch to watch-specific outlets, not GQ or NYT.
  6. Add schema.org Product markup — Structured data (price, brand, creator, availability, reviews) helps answer engines parse products. This compounds with the existing schema recommendation.
  7. Update content quarterly for freshness — Answer engines favor recent content for commercial prompts. Include current year in descriptions and update review counts.

Don't build content for a phantom business line. This should only become a Phase 2 priority if the founders confirm they want to pursue B2B design work. If yes, a /design landing page with portfolio examples and a contact form is the minimum viable SEO play.

11 Opportunities, Ranked by Impact

  • Add alt text to every product image AUTOMATED
    Product images currently have empty or auto-generated alt attributes. Adding descriptive alt text with artist names, watch names, and visual descriptions makes every image findable in Google Image Search. This is the single highest-impact SEO fix that requires zero visual changes. Automation built: Provide alt text in the JSON template and the script updates all images across all products in one run.
    Effort: Write the alt text descriptions (~1 hour). Script handles the Shopify updates (~2 minutes).
  • Write custom meta descriptions for every product page
    Auto-generated meta descriptions from thin product copy produce search results like "A beautiful timepiece from Mad Hudson Watches" — generic, keyword-poor, and indistinguishable from competitors. Custom meta descriptions with artist name, edition size, and "limited edition artist watch" phrasing directly improve click-through rates from search results. Template: "[Watch Name] — a limited edition watch designed by [Artist] for Mad Hudson. [One line about design]. [N] pieces. Free shipping." Aim for 150-160 characters.
    Effort: 3-5 min per product in Shopify admin → Products → Search engine listing preview. Entire catalog: ~1 hour.
  • Update page titles to include artist names
    Current auto-generated titles follow the pattern "[Watch Name] — Mad Hudson Watches." Adding artist names and "limited edition" creates richer, more clickable search results: "[Watch Name] by [Artist] — Limited Edition Artist Watch | Mad Hudson." This is the most prominent text in search results and directly affects ranking for artist-name queries.
    Effort: 2-3 min per page in Shopify admin → Products → Search engine listing preview → Page title. Entire catalog: ~30 min.
  • Enable Shopify bot filtering
    Free since Oct 2025. Current analytics show 12,983 sessions with only 8 conversions. Some of that traffic likely comes from Brad's social posts, but bot amplification inflates those days well beyond organic reach. Enabling bot filtering reveals the true conversion baseline (~0.85%) and makes all future analytics decisions trustworthy. Without this, every data-driven decision is built on numbers that mix real interest with bot noise. Next step: Cross-reference high-traffic days with Brad's Instagram/Meta posting history to separate genuine social-driven traffic from bot amplification.
    Effort: 5 min in Shopify admin → Analytics → Settings → toggle "Exclude known bots"
  • Add UTM parameters to Instagram link-in-bio and email campaigns
    179 orders ($28.3K — 62% of all revenue) have no tracked source. Instagram's in-app browser strips referrer data, so some of these "unattributed" orders may actually be Brad's Instagram traffic. Adding UTM parameters (?utm_source=instagram&utm_medium=social&utm_campaign=linkinbio) to the link-in-bio URL and any email links solves the attribution blindspot within a month. This costs nothing and tells you whether Instagram is your #1 revenue channel — which changes everything about where to invest next.
    Effort: 10 minutes. Update the link-in-bio URL and any email template links with UTM parameters. Shopify tracks these automatically.
  • Expand product descriptions from 1-2 sentences to 400-500 words AUTOMATED
    Current product pages have ~30 words of description — far below the 400-500 word target that performs well for SEO. Each product page should include: expanded product description (100-150 words), About the Artist section (100-150 words), Design Intent (75-100 words), detailed Specifications (50-75 words), and Edition Details (25-50 words). Automation built: Once descriptions are written, the script pushes all 6 products to Shopify in one run — titles, descriptions, metadata, and artist info together.
    Effort: Write the content (~45 min per product). Script handles the Shopify updates (~2 minutes for all 6). Validation checks word counts and meta description length before deployment.
  • Add JSON-LD schema markup for artist attribution
    No competitor (Mr Jones, Swatch x Art, Baltic, Brew, Anicorn) has structured data for artist/creator attribution on product pages. Adding JSON-LD with creator (Person), isLimitedEdition, material, and brand.description tells Google exactly what these products are and who made them. This enables rich search results, Knowledge Panel connections for artists, and defines how Google categorizes artist-collaboration watches. First-mover advantage in how this category gets indexed.
    Effort: Low-Medium — template the JSON-LD once in the product.liquid theme file (~2 hours), then it auto-populates for every product using Shopify's product fields.
  • Add money-back guarantee badge above Add to Cart
    Addresses the primary trust opportunity for an independent brand without widespread recognition. At ~$200, first-time buyers from search need reassurance. A "30-day money-back guarantee" badge placed above the Add to Cart button reduces purchase anxiety. This is a small theme edit — one HTML element and a few lines of CSS. The guarantee policy likely already exists; this just makes it visible at the decision point.
    Effort: 1 hour (small theme edit: add badge HTML + CSS to product template)
  • Enable Shop Pay Installments
    Reframes $200 as "$50/month." At this price point, installments aren't about affordability — they're about reducing perceived commitment for first-time buyers from search who don't yet trust the brand. Shop Pay is already integrated into Shopify; this is a settings toggle that adds "4 interest-free payments of $50" messaging below the price. No theme changes needed.
    Effort: 30 min (Shopify Payments settings toggle)
  • Verify and fix heading hierarchy (H1-H6) across all pages
    Shopify themes sometimes use heading tags for visual styling rather than semantic structure — producing pages with multiple H1 tags or skipped heading levels. Each page should have exactly one H1 (the product/page name), with H2s for major sections and H3s for subsections. Check by viewing page source and searching for heading tags. Any fixes are theme-level edits (changing an <h1> to a <div> in the header, for example).
    Effort: 1-2 hours to review all page types + fix in theme templates
  • Rename future image uploads with descriptive filenames
    Current image filenames are generic (IMG_4532.jpg, product-photo-1.png). While existing Shopify images can't be renamed, all future uploads should follow the pattern mad-hudson-[watch-name]-[artist]-[view].jpg. Google uses filenames as a signal for image search ranking. Combined with proper alt text, descriptive filenames make product images discoverable for queries like "[artist name] watch design."
    Effort: Ongoing habit — 30 seconds per image before uploading
