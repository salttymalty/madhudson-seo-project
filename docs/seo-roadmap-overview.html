H1 Hierarchy Fix — site-wide

The site's H1 tag is currently on the animated logo GIF. Google thinks the most important heading on every page is an image file. The fix is a Liquid template edit — zero visual change.

Before (current Liquid)
<!-- Logo image incorrectly wrapped in H1 --> <h1 class="site-header__logo"> <a href="/"> <img src="mad-hudson-logo.gif" alt="Mad Hudson"> </a> </h1>
After (corrected)
<!-- Logo is a div, H1 is the real page heading --> <div class="site-header__logo"> <a href="/"> <img src="mad-hudson-logo.gif" alt="Mad Hudson Watches"> </a> </div> <h1 class="visually-hidden">Mad Hudson Watches</h1>

The visually-hidden H1 is invisible to visitors but readable by search engines. Every page now has a semantically correct heading hierarchy. Will be tested on staging first.

Schema Markup — Product + Artist (future phase)

This invisible JSON-LD block tells Google exactly what each product is. It enables rich results (star ratings, price, availability) directly in search listings. Zero competitors have artist/creator schema.

<script type="application/ld+json"> { "@context": "https://schema.org", "@type": "Product", "name": "{{ product.title }}", "brand": { "@type": "Brand", "name": "Mad Hudson Watches" }, "creator": { "@type": "Person", "name": "{{ product.metafields.custom.artist_name }}" }, "offers": { "@type": "Offer", "price": "{{ product.price | money_without_currency }}", "priceCurrency": "USD", "availability": "https://schema.org/InStock", "shippingDetails": { "shippingRate": { "value": "0" } } }, "isRelatedTo": { "@type": "VisualArtwork", "name": "Limited Edition of 100" } } </script>

One template in the Liquid product layout. Shopify variables auto-fill per product. This is how search results get price, availability, and "limited edition" badges without any manual work per product.

What "Working" Looks Like

Realistic expectations for when changes become visible

Weeks 1–2

No visible change

Google is recrawling. Changes are submitted but not yet reflected in rankings or impressions. This is normal.

Weeks 3–4

Impressions start ticking up

Search Console shows MH pages appearing in more results. New queries may surface. Clicks may lag behind impressions.

Months 2–3

CTR improves

Better meta descriptions and title tags translate to higher click rates on the same impressions. If schema markup is live, rich snippets amplify this.

Months 3–6

Revenue signal

Organic search revenue starts growing above the $917/mo baseline. Artist-specific queries appear. The growth model projects $25K–$35K addressable upside — even 25% capture ($6–9K/yr) is a meaningful lift on a $52K base.
